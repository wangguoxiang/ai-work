设备绑定查询 · t_bind_log

按 VIN 与时间窗口查询处于绑定状态的设备(状态重叠语义,绑定段在内存配对,兼容 MySQL 5.7)

请输入查询条件后点击"查询"
说明: t_bind_log 是操作流水表,无独立 bind/unbind 字段。 后端按 (vin, tid) 分组、按 op_time 排序,在内存里把相邻的"绑定(0) → 解绑(2)"配对成一段; 若绑定后无解绑,则该段解绑时间为空(至今绑定)。
批量: 支持一次查询多个 VIN(每行一个,或用逗号/空格/分号分隔),结果按绑定时间排序、客户端分页展示。
合并去重: 若同一设备未解绑又被重新绑定,只算一次,并取较早的绑定时间为准。
筛选规则(状态重叠): 绑定开始 ≤ 窗口结束 且(未解绑 或 解绑 ≥ 窗口开始),即只要绑定关系与查询窗口有交集就返回。
